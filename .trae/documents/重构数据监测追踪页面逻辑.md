# 数据监测追踪重构计划：基于“创作计划”与“时间维度”的效果追踪

根据您的需求，我们将把“数据监测”模块从通用的品牌监测，转型为 **“创作计划效果追踪”**。核心逻辑是：**监测是计划的后续**，每一项数据指标都应归因于具体的意图（Intent）和计划（Plan），并随时间推移展示变化趋势。

## 1. 核心架构变更

### A. 关联“创作计划” (Plan-Centric Context)

不再展示孤立的“品牌能见度”，而是展示 **“已选计划（Plan）”** 的执行效果。
*   **计划选择器**：页面顶部增加下拉框，选择要监测的具体创作计划（如：“2025春节电车长途评测”）。
*   **意图关联**：自动显示该计划对应的核心意图（如：“20万电车推荐”）。

### B. 引入“时间维度”趋势 (Time-Series Analysis)

数据展示将以 **时间轴** 为核心，对比“计划执行前”与“执行后”的变化。
*   **趋势图**：展示该意图下，品牌在各大模型中的 **AI 推荐率 / 提及率** 随时间的变化曲线。
*   **关键事件点**：在时间轴上标记“内容发布”的时间点，直观展示优化动作带来的数据提升。

### C. 列表重构：意图驱动的监测详情

表格列将从通用的“问题分类”改为具体的 **“客户选择意图”** 及其衍生问题。

| 原有列 | **新列 (New Columns)** | 说明 |
| :--- | :--- | :--- |
| 问题分类 | **核心意图 & 衍生问题** | 比如核心意图是“20万电车”，衍生问题是“20万电车续航排行” |
| 搜索热度 | **当前排名/推荐率** | 实时监测数据 |
| 品牌漏出 | **环比变化** | 较上周/计划执行前的涨跌幅 |
| 负面风险 | **最新 AI 评价摘要** | 大模型对该意图的最新观点摘要 |

## 2. 具体修改步骤

1.  **重构** **`DataMonitoring.tsx`** **页面结构：**
    *   **Header**: 增加“计划选择器” (Select Plan)。
    *   **KPI Cards**: 展示该计划的总体 ROI（如：总曝光提升、AI 推荐率增长）。
    *   **Chart Section**: 实现“意图渗透率趋势图”，支持切换不同时间粒度（周/月）。
    *   **Detail Table**: 实现“意图监测详情表”，列出该计划覆盖的所有长尾意图及其当前表现。

2.  **Mock 数据联动：**
    *   构造模拟数据，体现“发布内容后，数据显著提升”的趋势，以验证 GEO 的有效性。

这一重构将打通“洞察 -> 计划 -> 监测”的闭环，让用户清晰看到“我做的计划（发的内容）到底有没有用”。
